Category one

Getting started

The basics: what ChemSearch is, where it runs, and how to get the app onto a device quickly.

What is ChemSearch?

ChemSearch is a native Android chemistry reference app. It helps you search compounds, inspect structures, review safety data, copy identifiers, and use built-in chemistry tools from one interface.

Do I need an account to use it?

No. The app does not require a sign-in flow or account before you can search compounds or use the tools.

What Android versions are supported?

The app targets Android 8.0 and newer, which corresponds to API level 26 and above.

Can I use ChemSearch on a tablet?

Yes. ChemSearch is designed for Android, so it can run on phones and tablets that meet the platform requirements. Layout and touch behavior remain usable on larger screens as well.

Where should I download the APK from?

The safest source is the official GitHub Releases page for the project. That is where new builds are published and versioned.

Does the app work fully offline?

Compound search and live reference data depend on internet access because the app talks directly to public APIs. Cached results, saved favorites, and some local workflows still help when you revisit something you already opened.

Category two

Search and data

How search behaves, which sources are involved, and why some compound pages may vary depending on source coverage.

What can I search by?

You can search by compound name, CAS number, or PubChem CID. That makes it flexible enough for classroom use, lab work, and quick lookups from documentation.

Which data sources does ChemSearch use?

PubChem is the main source for compound records, structures, autocomplete, identifiers, and safety-related data. Wikipedia is used as a fallback for plain-language descriptions when PubChem entries are sparse.

Does CAS search work directly?

Yes. CAS numbers are part of the search flow, so you can search with the identifier you already have instead of translating it into another field first.

Why do some descriptions look different from one compound to another?

Description quality depends on source coverage. Some compounds have strong PubChem summaries, while others rely on fallback plain-language descriptions when the main record is lighter.

Can I copy identifiers like SMILES, InChI, and CAS?

Yes. ChemSearch is designed so commonly reused identifiers can be copied directly from the result screen without extra navigation.

Does the app cache results?

Yes. There is a local cache for repeat lookups, which helps previously viewed compounds load faster and makes recent work easier to revisit.

Category three

Features and tools

What the app can do once you find a compound, plus what is included in the built-in chemistry toolkit.

Can I view compounds in 3D?

Yes. ChemSearch includes a native 3D viewer, and compound pages can switch from the 2D structure into an interactive 3D model when supported.

Can I export structure files or images?

Yes. The app supports exports such as SDF and PNG so you can move structures into other workflows without re-searching elsewhere.

How many built-in tools are included?

ChemSearch includes 11 built-in chemistry tools, covering reference workflows, stoichiometry, gas law calculations, dilution math, reaction scaling, and more.

Does the app include elemental analysis and formula details?

Yes. Compound pages surface formula-level information and elemental breakdown where the source record supports it.

Are GHS hazards and signal words included?

Yes. ChemSearch pulls structured safety information such as signal words, GHS pictograms, and hazard-related details from PubChem where available.

What happens when a record is missing some data?

The app can still present the fields and views that are available for that compound. Some sections may be less complete when the upstream data source does not provide them.

Category four

AI, privacy, and building from source

The questions that usually come up around API keys, permissions, local storage, and contributing to the project.

Do I need an API key for the app itself?

No. Core use of the app does not require an API key. You only need one if you explicitly enable optional AI summaries.

Which AI providers are supported?

The app supports optional AI summaries using Gemini and Groq. Those providers are not required for the main reference workflow.

Does ChemSearch send my API key through its own backend?

No. The project is built around direct integrations. Keys stay on your device, and requests go directly to the provider you enable.

What permissions does the app request?

The key permissions are related to internet access, optional update notifications, and export support on older Android versions where storage access behaves differently.

Can I build ChemSearch from source?

Yes. The repository is public and can be opened in Android Studio. The current documented build baseline is Android Studio Hedgehog or newer.

Where should I report a bug or request a feature?

The best place is the GitHub repository, especially the issue tracker. That keeps bug reports, suggestions, and follow-up discussion in one public place.
